# 🚀 mofuri (Next-Gen Static Site Generator)
WordPressの運用性と、Astro以上の静的パフォーマンスを、Bunネイティブで。
mofuriは、現在のWeb制作における「開発の難解さ」と「運用の重さ」を解消するために生まれた、Bunベースの超軽量・高速静的サイトジェネレータ（SSG）です [cite: 13, 111, 181]。


## 🌟 プロジェクト概要
mofuriは、「立ち上げればWordPress（圧倒的GUIの容易さ）、ビルドすればAstro以上（極限の超軽量・最小構成ファイル）」という「いいとこ取り」を唯一実現するツールです [cite: 180, 181]。

    100% Static Output: ビルド後の成果物には動的なランタイムや管理画面は1ミリも残りません。世界最速・ハッキングリスクゼロの静的な芸術品（HTML/CSS/JS）のみを出力します [cite: 13, 168]。
    React-free / Bun-native: 重厚なReactや巨大なnode_modulesを排除。Bunの標準JSXトランスパイラをハックし、コアシステムを数百〜1,000行程度の超軽量な規模で実現しています [cite: 1, 2, 164]。
    DB-less Architecture: データベース（MySQL等）は一切不要。すべてのデータはフロントマター付きのMarkdownファイル（.md）として管理されます [cite: 156, 157]。
    WordPress-like DX: テーマ開発者はWordPressのような直感的なルーティング（index.mofuri, single.mofuri）で開発でき、ノンプログラマーはローカルのGUIエディタで記事を書けます [cite: 13, 205, 206]。

## 📁 フォルダ構成
mofuriプロジェクトは、無駄を削ぎ落とした以下の5つのファイル種別（ts, tsx, .mofuri, md, yaml）だけで構成されます [cite: 4, 14, 28]。
```
.
├── .mofuri/             # コアシステム（CLIによって生成される実行ユニット） [cite: 77, 84]
├── pages/               # 固定ページ・ルーティング定義 (.mofuri) [cite: 76, 214]
├── posts/               # 記事データ (.md) [cite: 76, 152]
├── themes/              # テーマファイル（CSS、TSXコンポーネント） [cite: 77, 214]
├── plugins/             # 拡張機能 (TSX, manifest.yaml) [cite: 34, 35, 77]
├── public/              # 静的アセット（画像、faviconなど） [cite: 77, 227]
└── mofuri.yaml          # サイト設定ファイル [cite: 50, 75, 230]
```

## 🛠 CLIワークフロー
mofuri-cli を通じて、開発からビルド、プラグイン管理までを完結させます [cite: 58, 230]。

    mofuri init: プロジェクトを初期化し、.mofuri/ コアフォルダを展開します [cite: 173]。
    mofuri dev: ローカルサーバーを起動。同時に http://localhost:3000/mofuri-admin にGUI管理画面を提供します [cite: 224, 235]。
    mofuri build: 全記事とテーマを静的コンパイル。画像軽量化やCSS結合を行い、dist/ フォルダへ出力します [cite: 228, 229, 231]。
    mofuri add <plugin-id/url>: 公式リポジトリまたはGitHub URLからプラグインをダウンロードし、有効化します [cite: 59, 63, 64]。

## ⚙️ Coreの仕組み (Build Pipeline)
mofuri Coreは「md ➔ 中間HTML ➔ 最終HTML」の2段階コンパイルパイプラインを採用しています [cite: 101, 102]。

    データ解析: .md ファイルのフロントマターをパースし、本文をHTML（カスタムタグ <m-plugin> を含む）に変換します [cite: 11, 15, 120]。
    静的評価: .mofuri テンプレートの <script type="server"> をBun上で実行し、変数を評価します [cite: 11, 16, 190]。
    JSX置換: テンプレート内のカスタムタグを対応する .tsx プラグインへ流し込み、BunネイティブのトランスパイラでピュアなHTMLへと文字列化します [cite: 15, 16, 104, 105]。
    アセット結合: インポートされた全てのCSSを @layer で結合し、クライアント用JSと共に最小化（Minify）して出力します [cite: 48, 149, 150, 229]。

## 🔌 拡張機能 (Theme & Plugin)
### 🎨 テーマの仕組み (.mofuri)
テーマは画面全体の「器（うつわ）」を定義します。VueやSvelteに馴染んだ3分割構文を採用しています [cite: 192, 211]。
```svelte
<!-- themes/my-theme/pages/single.mofuri -->
<script type="server">
  // ビルド時に実行されるサーバーサイドロジック
  const { title, date } = post;
</script>

<template>
  <h1>${title}</h1>
  <m-github-card user="sofia-gros" />
  <article>${post.content}</article>
</template>

<script>
  // クライアント側で動作する最小限のJS
  console.log("mofuri Loaded!");
</script>
```

### 🧩 プラグインの仕組み (.tsx)
プラグインは、ロジックと部品を担当します。「必須ID規格（Required ID）」に基づき、Materialize CSSを参考にした共通部品を型安全に提供します [cite: 132, 140, 142]。

    コンポーネント・シャドウイング: テーマの components/ フォルダに同名の .tsx を置くことで、コア標準の見た目を自由にジャック（上書き）できます [cite: 218, 219, 220]。

## 📑 API記述テンプレート (manifest.yaml)
プラグインは manifest.yaml でその振る舞いを定義し、Coreが主導権を握ったまま安全に実行されます [cite: 34, 35, 36]。
```yaml
# plugins/my-plugin/manifest.yaml
id: my-github-card
version: 1.0.0
author: sofia-gros
entry: index.tsx
permissions:
  - fs:readPage   # 許可されたAPIフックを定義 [cite: 30, 35]
```

## 📝 ユーザー・開発者の役割分担
mofuriは、1つのシステムでありながらユーザーの技術スタックに合わせた2つの世界線を提供します [cite: 92, 115, 187]。

    執筆者（一般ユーザー）: GUI管理画面からMarkdownを意識せず記事を書き、WordPress級の手軽さでサイトを運営 [cite: 92, 115, 153]。
    開発者（ギーク）: TSX/TypeScriptをフルに活用し、型予測を効かせながら超軽量なテーマやプラグインを開発 [cite: 184, 185, 200]。

🔒 セキュリティと整合性

    完全DBレス: ハッキング対象となるDBが存在しないため、SQLインジェクション等の攻撃が根本から不可能です [cite: 47, 48, 157]。
    プラグイン統治: 非公式プラグインはGitHubからの直接インストールに限定し、ユーザーがリスクを確認した上でダウンロード・実行する明示的な主導権構造を取ります [cite: 23, 36]。
